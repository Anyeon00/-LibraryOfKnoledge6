1. 검증 로직 (HTML 폼에 입력값이 잘못된 경우)
-1. 타입 검증
ex) 가격, 수량에 문자가 들어오면 검증 오류

-2. 필드 검증
ex) 상품명 : 필수, 공백X
가격 : 1000원 이상, 1백만원 이하
수량 : 최대 9999

-3. 특정 필드의 범위를 넘어서는 검증(특정 상황 검증?)
ex) 가격 * 수량 합은 10,000원 이상

2. 검증 로직 실패시 오류 메시지

Version 1 ~ 4 설명


[검증 직접 처리 - 소개]
화면 흐름 (예시상황)
상품 등록 페이지 -> 상품저장 등록(성공) -> 상품 상세 페이지
상품 등록 페이지 -> 상품저장 등록(실패) -> 상품 등록 페이지(+오류결과)//redirect



[검증 직접 처리 - 개발]
Version1 (직접 다 코딩)
//컨트롤러 메서드에서
//1. 검증 오류를 보관
Map<String, String> errors = new HashMap<>();

//2. 검증 로직
// item가격이 null이거나 범위를 벗어난 경우
if (item.getPrice() == null || item.getPrice() < 1000 || item.getPrice() > 1000000) {
    errors.put("price", "가격은 1,000 ~ 1,000,000 까지 허용합니다");
 }

//3. 검증에 실패하면 다시 입력 폼으로
if (!errors.isEmpty()) {
    model.addAttribute("errors", errors);
    return "validation/v1/addForm";
}

//4. 이하로는 성공 로직

//5. html파일에서 타임리프로 오류메시지 출력하도록 수정작업(강의 자료 참고)

* 이 방법으로 타입 오류는 처리 불가능
* 스프링이 제공하는 기능을 알아보자
